How Mantri works

Introduction

Mantri is a Dependency Management System based on [Google's DepsWriter tool][depswriter]. The concept is that each file provides a namespace and can require any number of other namespaces

goog.provide('app.view.about');

The goog.provide() function ensures the existence of the JavaScript object structure indicated by its argument. It checks to see whether each object property in the path expression exists, and if it doesn't exist it initializes it.

The above function call is equivalent to:

app = app || {};
app.view = app.view || {};
app.view.about = app.view.about || {};

Because goog.provide() only initializes properties if they don't already exist, it never overwrites properties.

Using namespaces can be a style that you don't find to be of your taste. Mantri does not require that you follow this style. All you need to do is ensure that you uniquely identify your files with the goog.provide() method. Read more about the [goog.provide() and goog.require() functions][googrequire-wiki].
