The Configuration File

The mantriConf.json file is where you declare your vendor depedencies, build directives and more.

Show Me!

The full configuration with default values.

{
  "baseUrl": "js/",
  "libs": {},

  "deps": {
    "src": "js/",
    "root": "./",
    "dest": "deps.js"
  },

  "build": {
    "input": "js/app.js",
    "dest": "dist/app.min.js",
    "outputWrapper": "",
    "exclude": [""]
  }
}

Not For Production!

The mantriConf.json file should not exist in your production environment. So make sure you remove it at deployment. To define another location for the file use the data-config attribute in the <script> element:

<script data-config="js/config/mantriConf.json" src="js/libs/mantri.web.js"></script>

Options

baseUrl

Type: string Default: current directory

The base url path relative to your document root.

Absolute Paths

For cases where you need absolute paths feel free to use /

Using ./ means literally the current path. Helps if your application lives under a folder.

libs

Type: Object Default: None

Define all your third-party dependencies in this object. the key of the object is the lib's id and the value is the path to the file.

The key used to id the lib does not carry any significance beyond the config file itself.

deps.src

Type: string Default: ./

The source to scan for goog.provide and goog.require statements, essentialy the path to your web application.

The source must be a string representing an actual folder. It cannot be an array or use globbing patterns.

Be careful when defining a source. If the source is different from your document (web) root, you will need to define the documentroot using the root option.

deps.root

Type: string Default: ./

The root option defines the Document (web) Root directory of your website. It is required if you define the source option.

deps.dest

Type: string Default: deps.js

The file you wish the generated deps file to be saved at.

build.input

Type: string Default: Required

Define the path and filename to your main file that bootstraps your application. The path should be relative to where the Gruntfile.js is. The filename should include the .js extension.

build.dest

Type: string Default: app.min.js

Define the destination output file.

build.outputWrapper

Type: string Default: none

Define a wrapping string for your build application. Use the keyword <%=output%> to position the contents of your app:

"build": {
  "outputWrapper": "(function(){<%=output%>}).call(this);"
}

This will wrap your application in an IIFE invoked using the Global context. The string passed to the outputWrapper option will be evaluated using Underscore's template method.

build.exclude

Type: Array of strings Default: None

Define the libraries that you want to exclude from bundling. This is useful when you want to load popular libraries from public CDN hosts.

Use the key part of the libs hash to refer to libs you want excluded.
